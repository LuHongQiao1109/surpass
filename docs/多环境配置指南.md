# Vite + React 多环境配置指南

## 一、环境说明

| 环境 | 模式 (mode) | 环境文件 | 用途 |
|------|------------|---------|------|
| 开发环境 | `development` | `.env.development` | 本地开发调试 |
| 测试环境 | `test` | `.env.test` | 测试服务器部署 |
| 生产环境 | `production` | `.env.production` | 正式生产环境 |
| 本地环境 | `local` | `.env.local` | 个人本地配置（不提交） |

---

## 二、配置文件

### 2.1 开发环境 `.env.development`

```env
# 开发环境配置
VITE_APP_TITLE=React App - Dev
VITE_API_URL=http://localhost:3000/api
VITE_APP_ENV=development
```

### 2.2 测试环境 `.env.test`

```env
# 测试环境配置
VITE_APP_TITLE=React App - Test
VITE_API_URL=https://test-api.example.com/api
VITE_APP_ENV=test
```

### 2.3 生产环境 `.env.production`

```env
# 生产环境配置
VITE_APP_TITLE=React App
VITE_API_URL=https://api.example.com/api
VITE_APP_ENV=production
```

### 2.4 本地环境 `.env.local`（可选，不提交到 Git）

```env
# 本地环境配置（复制 .env.example 修改）
VITE_APP_TITLE=React App - Local
VITE_API_URL=http://localhost:8080/api
VITE_APP_ENV=local
```

## 三、Vite 配置

### vite.config.ts

```typescript
import { defineConfig, loadEnv } from 'vite'
import react from '@vitejs/plugin-react'
import path from 'path'

export default defineConfig(({ mode }) => {
  // 加载对应环境的环境变量
  const env = loadEnv(mode, process.cwd(), '')

  return {
    plugins: [react()],
    resolve: {
      alias: {
        '@': path.resolve(__dirname, './src'),
      },
    },
    server: {
      port: 5173,
      // 开发环境代理配置
      proxy: {
        [env.VITE_API_URL]: {
          target: env.VITE_API_URL,
          changeOrigin: true,
          rewrite: (path) => path.replace(new RegExp(`^${env.VITE_API_URL}`), ''),
        },
      },
    },
    build: {
      // 生产环境配置
      outDir: mode === 'production' ? 'dist' : `dist-${mode}`,
      sourcemap: mode === 'development',
    },
  }
})
```

### 配置说明

| 配置项 | 说明 |
|--------|------|
| `loadEnv(mode, process.cwd(), '')` | 加载对应环境的 `.env` 文件 |
| `mode` | 当前运行模式（development/test/production） |
| `env.VITE_API_URL` | 从环境文件读取的 API 地址 |
| `server.proxy` | 开发环境代理，解决跨域问题 |
| `build.outDir` | 不同环境输出到不同目录 |
| `build.sourcemap` | 仅开发环境生成 sourcemap |

## 四、npm 脚本配置

### package.json

```json
{
  "scripts": {
    "dev": "vite --mode development",
    "dev:test": "vite --mode test",
    "dev:local": "vite --mode local",
    "build": "vite build --mode production",
    "build:test": "vite build --mode test",
    "build:dev": "vite build --mode development",
    "preview": "vite preview --mode production"
  }
}
```

### 脚本说明

| 命令 | 说明 |
|------|------|
| `npm run dev` | 启动开发服务器（开发环境） |
| `npm run dev:test` | 启动开发服务器（测试环境） |
| `npm run dev:local` | 启动开发服务器（本地环境） |
| `npm run build` | 构建生产环境 |
| `npm run build:test` | 构建测试环境 |
| `npm run build:dev` | 构建开发环境 |
| `npm run preview` | 预览生产构建 |

## 五、在代码中使用环境变量

### 5.1 基本使用

```typescript
// 在组件中使用
console.log(import.meta.env.VITE_API_URL)
console.log(import.meta.env.VITE_APP_TITLE)
console.log(import.meta.env.VITE_APP_ENV)
```

### 5.2 封装配置模块

```typescript
// src/config/env.ts
interface EnvConfig {
  API_URL: string
  APP_TITLE: string
  APP_ENV: string
}

const envConfig: EnvConfig = {
  API_URL: import.meta.env.VITE_API_URL || '',
  APP_TITLE: import.meta.env.VITE_APP_TITLE || 'React App',
  APP_ENV: import.meta.env.VITE_APP_ENV || 'development',
}

export default envConfig
```

### 5.3 使用示例

```typescript
// src/views/testCode.tsx
import envConfig from '@/config/env'

function TestCode() {
  return (
    <div>
      <h1>当前环境：{envConfig.APP_ENV}</h1>
      <h2>应用标题：{envConfig.APP_TITLE}</h2>
      <p>API 地址：{envConfig.API_URL}</p>
    </div>
  )
}
```

### 5.4 环境变量类型定义

```typescript
// src/vite-env.d.ts
/// <reference types="vite/client" />

interface ImportMetaEnv {
  readonly VITE_API_URL: string
  readonly VITE_APP_TITLE: string
  readonly VITE_APP_ENV: string
}

interface ImportMeta {
  readonly env: ImportMetaEnv
}
```

## 七、环境变量命名规则

### 7.1 必须以 VITE_ 开头

Vite 只会暴露以 `VITE_` 开头的环境变量到客户端：

```env
# ✅ 会暴露到客户端
VITE_API_URL=http://localhost:3000

# ❌ 不会暴露（仅在 Node.js 中可用）
API_URL=http://localhost:3000
```

### 7.2 命名规范

```env
# 推荐：使用大写字母和下划线
VITE_API_URL=xxx
VITE_APP_TITLE=xxx

# 推荐：使用有意义的分组前缀
VITE_API_BASE_URL=xxx
VITE_API_TIMEOUT=xxx
VITE_FEATURE_FLAG_X=xxx
```

## 十、常见问题

### Q1: 环境变量不生效？

**解决方案：** 重启 Vite 开发服务器
```bash
# 停止当前服务，重新运行
npm run dev
```

### Q2: 如何查看当前环境变量？

```typescript
console.log(import.meta.env)
```

### Q3: 生产环境如何使用不同的 API 地址？

构建时使用不同的 mode：
```bash
# 测试环境
npm run build:test

# 生产环境
npm run build
```

### Q4: 如何在运行时动态切换环境？

Vite 的环境变量在构建时注入，无法在运行时切换。如需运行时切换：
- 使用配置文件 + 后端 API
- 使用构建参数注入

### Q5: 如何配置敏感信息？

敏感信息放入 `.env.local` 文件，不提交到 Git：
```env
# .env.local
VITE_API_KEY=your_secret_key
VITE_SECRET_TOKEN=xxx
```

## 十一、最佳实践

1. **使用 `VITE_` 前缀** - 只有带前缀的变量才会暴露到客户端
2. **提交 `.env.example`** - 提供配置模板给团队成员
3. **不提交 `.env.local`** - 个人配置和敏感信息不提交
4. **封装配置模块** - 统一管理环境变量访问
5. **添加类型定义** - 在 `vite-env.d.ts` 中定义类型
6. **使用环境变量分组** - 如 `VITE_API_xxx`、`VITE_FEATURE_xxx`
