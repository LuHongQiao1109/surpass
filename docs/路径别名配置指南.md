# Vite + React + TypeScript 路径别名配置指南

## 一、什么是路径别名

路径别名（Path Alias）是一种导入路径的简化方式，让你避免使用繁琐的相对路径。

### 对比示例

| 相对路径（不优雅） | 路径别名（优雅） |
|------------------|----------------|
| `import X from "../../components/X"` | `import X from "@/components/X"` |
| `import Y from "../../../utils/helper"` | `import Y from "@/utils/helper"` |
| `import Z from "./views/testCode"` | `import Z from "@/views/testCode"` |

---

## 二、配置步骤

### 步骤 1：安装 @types/node

Vite 需要使用 Node.js 的 `path` 模块来解析路径别名。

```bash
npm install -D @types/node
```

### 步骤 2：修改 vite.config.ts

```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import path from 'path'

export default defineConfig({
  plugins: [react()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
    },
  },
})
```

### 步骤 3：修改 tsconfig.json

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    },
    "ignoreDeprecations": "6.0"
  }
}
```

### 步骤 4：重启开发服务器

配置完成后，需要重启 Vite 开发服务器使配置生效。

```bash
npm run dev
```

## 三、配置原理解析

### 3.1 为什么需要两处配置？

| 配置文件 | 作用 |
|---------|------|
| `vite.config.ts` | Vite 构建工具在打包/运行时解析路径 |
| `tsconfig.json` | TypeScript 编译器在类型检查时解析路径 |

**两者缺一不可：**
- 只配 Vite：代码能运行，但 TypeScript 报错
- 只配 TS：类型检查通过，但运行时报错

### 3.2 baseUrl 和 paths 的含义

```json
{
  "baseUrl": ".",        // 基准目录：当前目录（tsconfig.json 所在目录）
  "paths": {
    "@/*": ["./src/*"]   // @/ 映射到 ./src/
  }
}
```

**映射关系：**
- `@/components/Button` → `./src/components/Button`
- `@/utils/helper` → `./src/utils/helper`
- `@/views/testCode` → `./src/views/testCode`

### 3.3 ignoreDeprecations 的作用

TypeScript 6.0 开始废弃 `baseUrl` 选项，但 Vite 项目仍需要使用它。添加此配置可抑制警告：

```json
"ignoreDeprecations": "6.0"  // 忽略 TypeScript 6.0 的废弃警告
```

> 未来 TypeScript 可能使用 `moduleResolution: "bundler"` 的新方式，但目前 `baseUrl + paths` 仍是最通用的方案。

## 四、常用别名模式

### 4.1 单一别名（推荐）

```json
{
  "@/*": ["./src/*"]
}
```

**优点：** 简单统一，所有 src 下的文件都用 `@/` 开头

### 4.2 多别名

```json
{
  "@/*": ["./src/*"],
  "@components/*": ["./src/components/*"],
  "@utils/*": ["./src/utils/*"],
  "@hooks/*": ["./src/hooks/*"],
  "@assets/*": ["./src/assets/*"],
  "@views/*": ["./src/views/*"]
}
```

**优点：** 语义更清晰
**缺点：** 配置较繁琐，重构时可能需要修改多处

### 4.3 组件专用别名

```json
{
  "@/*": ["./src/*"],
  "#": "./src/components"
}
```

使用效果：
```typescript
import Button from '#/Button'
```

## 五、使用示例

### 项目结构

```
src/
├── components/
│   └── Button/
│       ├── Button.tsx
│       └── Button.css
├── utils/
│   └── helper.ts
├── hooks/

│   └── testCode.tsx
├── assets/
│   └── logo.png
```

```typescript
import Button from '@/components/Button/Button'
```

## 六、常见问题

### Q1: 配置后 VSCode 没有类型提示？

**解决方案：** 重启 VSCode 或 TypeScript 服务
- `Ctrl+Shift+P` → "TypeScript: Restart TS Server"

### Q2: 配置后运行报错？

**解决方案：** 重启 Vite 开发服务器
```bash
# 停止当前服务，重新运行
npm run dev
```

### Q3: 可以使用其他别名符号吗？

可以，`@` 只是约定俗成的习惯，你也可以用 `~`、`#` 等：

```json
{
  "~/*": ["./src/*"]
}
```

```typescript
import X from '~/components/X'
```

### Q4: ESLint 报错？

如果使用 ESLint，可能需要配置 `import/resolver`：

```json
// .eslintrc.json
{
  "settings": {
    "import/resolver": {
      "typescript": {
        "project": "./tsconfig.json"
      }
    }
  }
}
```

## 七、完整配置文件

### tsconfig.json

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "useDefineForClassFields": true,
    "lib": ["ES2020", "DOM", "DOM.Iterable"],
    "module": "ESNext",
    "skipLibCheck": true,
    "moduleResolution": "bundler",
    "allowImportingTsExtensions": true,
    "resolveJsonModule": true,
    "isolatedModules": true,
    "noEmit": true,
    "jsx": "react-jsx",
    "strict": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "noFallthroughCasesInSwitch": true,
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    },
    "ignoreDeprecations": "6.0"
  },
  "include": ["src"],
  "references": [{ "path": "./tsconfig.node.json" }]
}
```

### vite.config.ts

```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import path from 'path'

export default defineConfig({
  plugins: [react()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
    },
  },
})
```
